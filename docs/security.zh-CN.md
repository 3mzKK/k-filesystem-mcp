# 安全说明

K Filesystem MCP 运行在全局本地文件系统模式。

这意味着它可以读取、写入、复制、移动和删除当前操作系统用户有权限访问的文件。这个设计适合个人本地 Codex 工作流，但不适合暴露成远程共享文件服务。

## 使用边界

- 只建议在可信的本地个人环境中使用。
- 不要暴露给远程客户端。
- 不要在不可信用户可以连接的环境中运行。
- 不要把它当成有隔离能力的沙箱。

## 高风险操作

以下操作建议先使用 `dryRun: true`：

- 覆盖已有重要文件。
- 删除用户数据或项目数据。
- 递归删除目录。
- 复制大型目录或来源/目标不明确的目录。
- 范围不清的批量编辑。

低风险、刚刚读过目标范围的小范围精确编辑，可以跳过 dry-run 以减少一次往返。

## 已有护栏

- `write_file` 覆盖已有文件必须显式 `overwrite: true`。
- 大文件被替换成明显更短内容时，`write_file` 会触发 major overwrite guard，除非显式 `allow_major_overwrite: true`。
- `write_file` 可选 `backup_existing: true`，但默认关闭，避免制造无意义备份文件。
- `remove_path` 默认 dry-run。
- 非空目录删除需要 `recursive: true`。
- 递归删除受 `max_entries` 限制。
- `copy_path` 默认拒绝已有目标；目录目标不会被隐式合并。真实递归目录复制在开始复制前会先受 `max_entries` 和 `timeout_ms` 约束做 preflight。

## 推荐习惯

- 删除前先确认路径足够具体。
- 复制/移动/删除时检查结构化状态字段。
- 遇到结构化 `code` 时，优先调整参数或缩小范围，不要盲目重试。
- 对长日志、长文档和记录文件，优先用 `edit_file` 做追加或局部修改，不要整文件覆盖。
