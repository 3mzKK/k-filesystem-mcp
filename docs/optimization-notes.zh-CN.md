# 优化记录

这份记录只保留已经对外可见、值得用户了解的优化方向，不展开内部调试过程。

## 已完成的主要优化

- 范围读取和尾部读取优化，减少不必要的整文件读取。
- 多文件批量读取优化，减少重复请求。
- 搜索输出优化，支持更省 token 的结果模式和分片续搜。
- 目录浏览优化，加入摘要、稳定排序和更紧凑的输出控制。
- 元数据查询优化，支持批量读取、`metadata_concurrency` 并发限制和更一致的结构化错误。
- diff 输出优化，避免超长行和超长 diff 占用过多上下文。
- edit_file 优化，加入锚点编辑、原始范围批量应用、结构化诊断和更短的失败信息。
- write_file / copy_path / move_file / remove_path 优化，加入覆盖护栏、预览、递归边界、耗时/状态字段和结构化错误。`copy_path` 现在会用 `max_entries` 和 `timeout_ms` 对真实递归目录复制做 preflight。
- edit/write/diff/search 的输出压缩，尽量只返回完成判断真正需要的信息。

## 设计原则

- 优先减少重复读取和重复失败。
- 优先让工具返回结构化状态，而不是让模型从大段自然语言里猜结果。
- 优先让文档明确边界，减少 Codex 选错工具。
- 高风险操作默认保守，低风险精确操作默认轻量。

## 使用建议

- 已知文件优先范围读，不要整文件读大文件。
- 已知多文件优先批量读。
- 搜索只要路径或计数时，不要拿完整内容。
- 编辑前先读最小目标范围，必要时再加锚点。
- 复制、移动、删除和覆盖前先判断风险，再决定是否 dry-run。
